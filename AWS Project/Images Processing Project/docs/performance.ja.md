# Performance Analysis

#### 🌐 言語 / Language / 语言

 英語: [performance.md](performance.md)                                               日本語: [performance.ja.md](performance.ja.md)

## テスト環境

- ランタイム: Python 3.12
- アーキテクチャ: x86_64
- メモリ割り当て: 256 MB
- トリガー: SQS イベント

テストは、複数の画像をS3にアップロードして実施しました。

## 観測メトリクス

CloudWatch メトリクス:

- 実行時間 (Duration)
- 呼び出し回数 (Invocation count)
- エラー回数 (Error count)
- メモリ使用量 (Memory usage)

## サンプル結果

(後日、実際の数値に置き換えてください)

- 平均実行時間: 約 382 ms
- 最大実行時間: 約 716 ms
- メモリ使用量: 約 45 MB

## 観測結果

- Lambdaのコールドスタートは、依存関係が軽量なため最小限に抑えられています。
- メモリ割り当ては実際の使用量よりも高く、最適化の余地があることを示しています。
- スループットはメッセージ量に応じて自動的にスケールします。

## 最適化の機会

- メモリを削減してコストを下げる
- バッチ処理の導入
- 同時実行数の制限を追加
- 予測可能なレイテンシのためにプロビジョニッドコンカレンシーを使用

## まとめ

このシステムは、イベント駆動型の負荷に対してスケーラブルなパフォーマンスを示しています。

パフォーマンスは主に画像サイズと処理の複雑さに依存します。

サーバーレス設計により、手動介入なしで自動スケーリングが可能です。
