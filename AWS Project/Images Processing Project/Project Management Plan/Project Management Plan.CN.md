## 2. 项目目标与范围

### 2.1 项目目标

- 实现图像自动上传与处理流程  
- 提高系统的可扩展性与稳定性  
- 降低人工运维成本  
- 确保数据访问的安全性  

### 2.2 项目范围

**包含**：
- 图像上传模块（API Gateway + Lambda + S3）  
- 图像处理模块（压缩、加水印）  
- 消息队列与事件驱动流程（SQS 解耦）  
- 数据存储与生命周期管理（S3 + DynamoDB）  
- 用户认证与权限控制（Cognito + IAM）  
- 处理完成通知（SNS）  

**不包含**：
- 前端界面（仅提供 API）  
- AI/机器学习图像识别  
- 视频处理  
- 本地部署或混合云方案

## 4. 假设与约束

### 4.1 假设

- 用户网络可正常访问 AWS 服务  
- 仅处理常见图像格式（JPEG、PNG）  
- 使用 AWS 免费套餐或低成本账号，无预算审批  
- Lambda 最大执行时间 15 分钟足够处理常规图像  

### 4.2 约束

- 不使用 EC2 或 Kubernetes，仅限无服务器服务  
- 不涉及 GPU 加速  
- 图像处理不支持超大文件（>1GB）  
- 项目为单人开发，无额外测试团队  

## 5. 架构设计概述

- S3：原始桶 + 处理后桶  
- API Gateway + Lambda：上传接口  
- SQS：异步解耦，削峰填谷  
- Lambda（处理函数）：压缩、水印  
- SNS：发送处理完成通知  
- DynamoDB：存储图像元数据（文件名、大小、处理时间、状态）  
- Cognito：用户认证 + 临时凭证  
- IAM + Security Boundary：最小权限隔离

## 7. 风险管理

| 风险                     | 影响 | 应对措施                                    |
| ------------------------ | ---- | ------------------------------------------- |
| 高并发导致系统压力       | 高   | SQS 削峰填谷，Lambda 并发预留               |
| 权限配置错误导致数据泄露 | 高   | IAM 最小权限 + Security Boundary + 定期审计 |
| 消息处理失败或丢失       | 中   | SQS 死信队列 + 重试机制 + 失败告警          |
| 存储成本过高             | 中   | S3 生命周期策略（标准 → IA → Glacier）      |
| Lambda 执行超时          | 低   | 限制图像尺寸，或拆分为多个步骤              |

## 8. 成本管理

- 计算成本：仅 Lambda 按调用次数和运行时长计费，空闲无成本  
- 存储成本：S3 分层 + DynamoDB 按需模式  
- 通知成本：SNS 每月前 100 万次免费  
- 队列成本：SQS 每月前 100 万次免费  
- 监控：CloudWatch 免费配额内足够  
- 优化动作：设置 AWS Budgets 告警，防止超支  

## 9. 质量管理

### 9.1 测试策略

| 测试类型 | 范围                          | 工具/方法               |
| -------- | ----------------------------- | ----------------------- |
| 单元测试 | Lambda 处理函数               | Python                  |
| 集成测试 | API → S3 → SQS → Lambda → SNS | AWS CLI                 |
| 安全测试 | IAM 权限、Cognito 临时凭证    | AWS IAM Policy          |
| 性能测试 | 模拟 100 并发上传             | Artillery / Locust      |
| 可观测性 | 日志、错误、延迟              | CloudWatch Logs + X-Ray |

### 9.2 验收标准

- 图像上传后 30 秒内完成处理（99% 请求）  
- 无 IAM 权限越界警告  
- SQS 死信队列在失败 3 次后正确接收消息  
- 所有 API 请求需经 Cognito 认证  

## 10. 监控与运维

### 10.1 监控指标

- Lambda 调用次数、错误率、持续时间  
- SQS 队列长度（可见 + 不可见消息数）  
- S3 存储桶大小与请求数  
- DynamoDB 读/写容量使用情况  
- CloudWatch 告警阈值：  
  - Lambda 错误率 > 5%  
  - SQS 积压 > 100 条消息持续 5 分钟  

### 10.2 日志与追踪

- 所有 Lambda 输出日志到 CloudWatch Logs  
- 启用 X-Ray 追踪关键请求链路  
- 定期导出日志到 S3 用于长期审计  

### 10.3 灾难恢复

- S3 跨区域复制（视成本决定）  
- DynamoDB 按需备份（每周一次）  
- 元数据可重建：通过扫描 S3 桶重新生成 DynamoDB 记录  

## 11. 安全策略

- 用户认证：Cognito 用户池 + 身份池  
- 临时凭证：通过 Cognito 获取 AWS 凭证，限制仅能上传到特定 S3 前缀  
- IAM 角色：  
  - API 调用角色：仅允许写入 S3 原始桶 + 发送 SQS  
  - 处理函数角色：仅允许读取原始桶、写入处理后桶、写入 DynamoDB、发布 SNS  
  - 安全边界：明确限制资源 ARN，不使用通配符 `*` 资源  
- 加密：S3 服务端加密（SSE-S3），DynamoDB 默认加密
