# AWSサーバーレス画像アップロード・処理システム

#### 🌐 言語 / Language / 语言

 🇺🇸English: [README.md](README.md)                                                🇯🇵日本語: [README.ja.md](README.ja.md)                                         🇨🇳中文: [README.CN.md](README.CN.md)

このプロジェクトは、AWSサーバーレスアーキテクチャ上に構築された、高性能でスケーラブルな画像アップロード・処理システムです。中核ロジックは3つの連携して動作するLambda関数で構成されており、セキュアなアップロードから非同期処理、結果通知、監視までのエンドツーエンド機能を実装しています。

## システムアーキテクチャ概要

**主な機能**

- **完全サーバーレスアーキテクチャ：** Lambda、S3、SQSなどのマネージドサービスをベースに、サーバー管理不要で自動スケーリング。
- **非同期疎結合設計：** SQS標準キューを使用してアップロードと処理のワークフローを分離し、負荷時のシステム信頼性と耐障害性を向上。
- **安全なストレージ分離：** 元画像と処理済み画像は別々に保存され、Cognito認証によりアップロードのセキュリティを確保。
- **プロダクショングレードの可観測性：** CloudWatchモニタリングとSNS通知と完全統合され、追跡とアラートが容易。
- **メタデータ管理：** DynamoDBを使用してすべての画像メタデータを保存し、追跡とクエリを容易に。

### Lambda関数の詳細説明

このシステムには、以下の3つの中核となるLambda関数が含まれています。

#### 1. Generate-Upload-URL Lambda

**機能**

- 認証されたCognitoユーザーに対して、セキュアなS3プリサインドアップロードURL（PUTメソッド）を生成します。
- 各ユーザーのアップロード画像を、それぞれ専用のディレクトリに自動分類します。

**入力/出力**

- **入力：** ユーザーID（Cognito User ID）は、API GatewayのJWTオーソライザー経由で自動的に渡されます。

- **出力：**

  ```json
  {
    "upload_url": "<PresignedURL>",
    "file_path": "user-uploads/<userId>/<uuid>.jpg"
  }

**セキュリティ機能**

- ログイン済みのCognitoユーザーのみが呼び出し可能。
- プリサインドURLの有効期間はデフォルト5分とし、悪用を防止。
- ディレクトリ分離によりユーザーデータのセキュリティを確保し、他人のファイルの上書きを防止。

#### 2. Image Processor Lambda

これはシステムの中核となる処理ユニットです。

**機能**

- SQS標準キューから画像アップロードイベントメッセージを受信します。
- 元のS3バケットから元画像をダウンロードします。
- Pillowライブラリを使用して画像処理（サムネイル生成、形式変換など）を実行します。
- 処理結果を指定されたターゲットS3バケットにアップロードします。
- 処理ステータスとメタデータ（画像ID、S3パス、ステータスなど）をDynamoDBテーブルに書き込みます。
- 処理結果通知（成功/失敗）をSNSトピックに送信します。

**バックエンド非同期処理の利点**

- **高並行処理：** Lambdaの自動スケーリング機能を活用し、トラフィック変動に対応。
- **疎結合と信頼性：** SQSがバッファ層として機能し、処理コンポーネントが障害を起こしてもタスクが失われない。
- **タスクの独立性：** 各処理タスクは分離されており、単一タスクの失敗がシステム全体に影響しない。

**出力例（DynamoDBに書き込み）**

```json
{
  "imageId": "uuid",
  "originalS3Key": "user-uploads/xxx/xxx.jpg",
  "processedS3Key": "processed/uuid.jpg",
  "status": "DONE",
  "lastModified": "2023-10-01T12:00:00Z"
}
```

#### 3. S3 Metrics Monitoring Lambda

**機能**

- 定期的（設定可能：1分、5分、1時間など）に指定されたS3バケットのサイズとオブジェクト数を統計します。
- カスタムメトリクス（例：`BucketSizeBytes`、`NumberOfObjects`）をCloudWatchに報告します。

**目的**

- **コスト監視とレポーティング：** ストレージの成長傾向を監視し、コスト最適化を支援。
- **プロアクティブなアラート：** メトリクスに基づいてCloudWatchアラームを設定（例：ストレージ容量のしきい値）。

**依存関係**

- `boto3` SDKのみを使用し、追加のサードパーティライブラリ依存はありません。

## 監視と運用

システムはCloudWatchと完全統合されており、包括的な監視が可能です：

- 各Lambda関数の実行ログ、実行時間、エラー率を表示。
- CloudWatchコンソールでカスタムメトリクス（`CustomS3`名前空間）に基づいてアラームを設定。
- SNS通知を介して処理の成功または失敗のメッセージを受信。
