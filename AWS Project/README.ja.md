## プロジェクト概要

このリポジトリは、AWSクラウドコンピューティング、サーバーレスアーキテクチャ、イベント駆動システム、およびクラウドセキュリティ管理における私の総合的なスキルを紹介しています。以下の2つのメインプロジェクトが含まれています。

1. 画像処理プロジェクト：ユーザーがアップロードした画像の自動処理、非同期タスクスケジューリング、および安全な保存。
2. セキュリティインシデント自動化・コンプライアンス監査プラットフォームプロジェクト：エンタープライズグレードのセキュリティ管理およびインシデント対応プラットフォーム。

これら2つのプロジェクトを通じて、AWSサービス統合、クラウドアーキテクチャ設計、サーバーレスコンピューティング、およびセキュリティ保護における実践的な能力を示します。

## 1. 画像処理プロジェクト

### プロジェクト目標

ユーザーが画像をアップロード → 自動処理 → 安全な保存

非同期タスクスケジューリングによるシステム拡張性の向上

Lambda、S3、SQS、およびCognitoの統合を示す。

### 主要コンポーネント

Lambda関数：画像アップロード、非同期処理、タスク通知。

S3：元画像と処理済み画像の保存。

SQS：非同期タスクキュー。

Cognito：ユーザー登録と認証。

IAM：最小権限のロール管理。

## 2. セキュリティインシデント自動化・コンプライアンス監査プラットフォームプロジェクト

### プロジェクト目標

エンタープライズグレードのAWSセキュリティシナリオをシミュレーション。

GuardDutyを構成して異常アクセスを監視。

IAMポリシーとロールを使用して権限を管理。

イベント駆動型のセキュリティ対応ワークフローを構築。

### 主要コンポーネント

GuardDuty：リアルタイムのセキュリティ脅威監視。

EventBridge：イベントルーティングとルールトリガー。

Lambda：自動化されたセキュリティインシデント対応。

IAM：ユーザーとロールの権限管理。

## 3. デプロイガイド

1. AWS CLIを構成し、アクセスキーを設定する。
2. S3バケットを作成する（画像処理プロジェクト用およびセキュリティログ用）。
3. IAMロールとポリシーを構成し、最小権限を確保する。
4. Lambda関数をデプロイし、トリガーを構成する。
5. SQSキュー（画像処理プロジェクト）またはEventBridgeルール（セキュリティプロジェクト）を構成する。
6. Cognitoユーザープールを設定する（画像処理プロジェクト）。
7. ワークフロー全体をテストし、画像処理とセキュリティインシデント対応が正常に行われることを確認する。

## 4. 技術スタック

AWSサービス：Lambda、S3、SQS、IAM、Cognito、GuardDuty、EventBridge、CloudWatch、SNS

プログラミング言語：Python 3.11+

ツール：draw.io

## 5. プロジェクトの特徴

フルスタックAWS能力を示す：サーバーレスアーキテクチャ + 非同期処理 + セキュリティ管理。

イベント駆動型の自動化ワークフローと高可用性アーキテクチャを実装。

セキュリティとアクセス制御を重視し、最小権限の原則を遵守。
